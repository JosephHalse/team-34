Comparison of Potential Devices

In theory the app could work on any android device such as a smart phone. The brief the client gave us specifically stated that it should be for the Epson Smart Glasses, so we favoured this as the final device that it should be designed for use on. In the situation described where the user is administering to a patient and using a perhaps unfamiliar piece of medical technology it would be smoother for them to use something which enables a clear view of what’s seen in-front of them, as well as the information related to the device. If using a handheld device like a smart phone they would have to keep looking away to read the information, by using the glasses the text would be clearly displayed in - front of them, merged with a real life view. This way it’s more of a true augmented reality and fulfils the need to be user friendly and innovative. 
Use of the smart glasses is a convenient way of interfacing with the app , leaving the user hands free to work.

Comparison of Different Image Recognition Application Programming Interfaces (APIs)

TensorFlow

Pros :

Offers multiple ways of doing the algorithm some of which a re suitable for working in real - time. Most well - known, therefore more documentation available to help with setting up.

Favoured by the client

Cons :

Complex to set up

OpenIMAJ

Pros :

Easier to implement, requires less code.

Cons :

Very limited documentation, has less features than TensorFlow

OpenCV

Pros :

Good documentation , appears to be popular and widely used, well known.

Cons :

Not proper deep learning

Therefore we initially proceeded to use TensorFlow, however we encountered problems with it as it performed far too slowly on the hardware making it almost unusable. We tested the android version of tensorflow (lite) on the glasses, having seen it perform well on a smart phone, but it ran too slowly to be practical, this may have been because the glasses have a low specification compared to a high end android phone for which TensorFlow is intended. We then downloaded Epson Moverio Software Development Kit (SDK) which is designed for use with this specific model of smart glasses, gaining access to the beta version which is available for developers via the Epson website.
