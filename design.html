Design

Key Functionalities Implemented

Object Recognition

Object recognition was implemented using the Fritz Machine Learning API. We decided upon Fritz due to is flexibility, compatability with API 22 and its deployment of custom models. We have the camera continually sending data to the Fritz API, the api then analyzes this data for our custom models. We created these custom models to be used by Tensorflow Lite as Fritz's custom model program is built upon Tensorflow. The API then anaylzes the bitmap that is sent and sends back the results. The way our object recognition works is that Fritz attempts to find the object most closely resembling the bitmap sent. It can only choose from the objects we have provided in the custom model package and will reply with the percentages of what it thinks the object is. If any object is above 70% then it will display the text saying that that particular object has been detected.

Custom Model Creation

By using a well-known open-source software library called Tensorflow Lite [1], we are able to achieve the above feature even with our custom models. It is a lightweight and ideal solution for mobile or embedded device, in our case, the Epson Smart Glasses. However, training a Tensorflow model can really take a long time if starting from scratch. Luckily, we could make use of Transfer Learning which is a process of retraining an existing pretrained model and produce a new model [2]. By figuring out the appropriate pretrained model in our case the 'mobilenet v1' and training parameters, simply by running a training script and an optimization script, we can get the model file in 30 minutes. We later had a lot of testing with the models, it really demonstrates the performance in a way that is fast and accurate. If a new object is considered to be added in the future, we also made it very easy to do so, which we put all training commands together into one script. Just set up the right path in the script and run it, your new model will be there for you. 

Voice Recognition

By making use of the IBM Watson Speech-to-Text API[3] , we are able to precisely transcribe human’s phrases, which enable users to control the app with voice instead of using hands or external controller to click buttons. In our project, users are able to skip from ‘the welcome page’ to ‘the object detecting page’ whenever the voice recogniser detects the word ‘next’ in a speech. Moreover, users can pull up instruction for the corresponding detected object. When users have finished reading the instruction, simply by saying 'go back' it will go back to 'the object detection page'. In a nutshell, this feature should deliver a good user experience to users and speeds up their process of doing tasks. Since we are using a free IBM cloud service account, it only provides us with a 100-minute-free Speech-to-Text service every months. If the app somehow doesn't follow the voice command, checking or updating the listening allowance is necessary 

PDF Viewer

This is an activity for an Android application capable of display any PDF file via the internet, inserted into the prototype. Methods for rendering a PDF file to image view were researched online and coded into a simple app. Rather than have it open a browser window in a separate application, or take them from local storage, the final version was integrated into the app, obtaining files from online and displaying them within the application.
